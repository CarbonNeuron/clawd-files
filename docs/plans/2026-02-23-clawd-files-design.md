## Overview

Clawd Files is a file hosting platform built for LLMs as first-class users. Think GitHub repos, but for temporary/permanent file sharing with API-first design. Buckets (like repos) hold files and folders, are owned by named API keys, and expire automatically.

## Architecture

**Server Components first.** All pages are React Server Components. Client components used only where interactivity is required (sortable CSV tables, markdown raw/rendered toggle, admin dashboard action buttons). Shiki runs server-side (zero client JS). ## Data Model

### `api_keys`
| column | type | notes |
|---|---|---|
| `key` | text PK | SHA-256 hash of raw key |
| `prefix` | text | first 8 chars of raw key â€” for admin identification |
| `name` | text | label, e.g. "SQL Buddy" (duplicates allowed) |
| `created_at` | integer | unix timestamp |
| `last_used_at` | integer | updated on each auth |

Raw key shown once on creation, never stored.

### `buckets`
| column | type | notes |
|---|---|---|
| `id` | text PK | 10-char nanoid |
| `name` | text | display name |
| `key_hash` | text | FK to `api_keys.key` â€” used for ownership auth |
| `owner` | text | display name (copy of `api_keys.name` at creation time) |
| `description` | text | nullable |
| `for_field` | text | nullable â€” displayed as "by X, for Y" |
| `created_at` | integer | unix timestamp |
| `expires_at` | integer | nullable = never expires |

**Key revocation policy:** When an API key is revoked, its buckets become read-only orphans. No key can manage them, but they remain publicly viewable and expire naturally.

### `files`
| column | type | notes |
|---|---|---|
| `id` | integer PK | autoincrement |
| `bucket_id` | text FK | cascade delete |
| `path` | text | e.g. `reports/q1.csv` |
| `size` | integer | bytes |
| `mime_type` | text | detected on upload |
| `created_at` | integer | unix timestamp |

Unique constraint on `(bucket_id, path)` â€” re-uploading a path replaces the file.

### File storage on disk
```
$DATA_DIR/
  db.sqlite
  files/
    {bucket_id}/
      {path...}
```

## Authentication

### Admin
- API endpoints: `Authorization: Bearer $ADMIN_API_KEY`
- Dashboard page: HMAC-SHA256 signed token in query param (`/admin?token=...`)
  - Token format: `base64(expires_at + "." + hmac_sha256(expires_at, ADMIN_API_KEY))`
  - Valid for 24 hours, non-renewable â€” generate a new link via API after expiry
  - Generated by `GET /api/admin/dashboard-link`
  - No session, no cookies â€” verify HMAC + timestamp on each request

### LLM API Keys
- `Authorization: Bearer <raw_key>`
- On each request: SHA-256 hash the key, look up in `api_keys`, update `last_used_at`
- Scoped to own buckets only (matched via `buckets.key_hash`)

### Auth Helper (`src/lib/auth.ts`)
Single `authenticate(request)` function returns:
- `{ type: 'admin' }` â€” for admin API key
- `{ type: 'key', keyHash: string, name: string }` â€” for LLM API key
- Throws on invalid/missing auth

### Route Protection

| Route | Access |
|---|---|
| `POST /api/keys` | Admin only |
| `GET/DELETE /api/keys` | Admin only |
| `GET /api/admin/dashboard-link` | Admin only |
| `/admin?token=...` | Valid HMAC token |
| `POST /api/buckets` | Any valid API key |
| `GET /api/buckets` | API key (own buckets only) |
| `GET/PATCH/DELETE /api/buckets/:id` | Owner key or admin |
| `POST /api/buckets/:id/upload` | Owner key or admin |
| `DELETE /api/buckets/:id/files` | Owner key or admin |
| `GET /api/buckets/:id/zip` | Public â€” no auth |
| `/{bucket_id}/...` (view/download) | Public â€” no auth |

## Content Negotiation & Routing

### Middleware (`src/middleware.ts`)
Runs on every request matching `/:bucket/:path+` (excluding `/raw/`, `/api/`, `/admin`, `/docs`, `/llms.txt`):
- `Accept` does **not** include `text/html` â†’ rewrite to `/raw/{bucket}/{path}`
- Otherwise â†’ pass through to `page.tsx` for HTML preview

The `/raw/` prefix is an explicit override that always returns raw bytes regardless of `Accept`.

### Route Map
```
src/app/
  page.tsx                              # Landing page
  docs/
    page.tsx                            # API docs (interactive, Stripe-style)
    api.md/route.ts                     # Raw markdown at /docs/api.md
  admin/
    page.tsx                            # Dashboard (HMAC token-gated)
  [bucket]/
    page.tsx                            # Bucket view (file tree + README)
    [...path]/
      page.tsx                          # File preview (HTML)
  raw/
    [bucket]/[...path]/
      route.ts                          # Always raw â€” streams file bytes
  api/
    keys/route.ts                       # POST, GET
    keys/[key]/route.ts                 # DELETE
    buckets/route.ts                    # POST, GET
    buckets/[id]/route.ts              # GET, PATCH, DELETE
    buckets/[id]/upload/route.ts       # POST multipart
    buckets/[id]/files/route.ts        # DELETE
    buckets/[id]/zip/route.ts          # GET â€” stream ZIP
    buckets/[id]/summary/route.ts      # GET plain-text (LLM-friendly)
    admin/dashboard-link/route.ts      # GET
    og/[bucket]/route.tsx              # GET â€” dynamic OG image
    og/[bucket]/[...path]/route.tsx    # GET â€” dynamic OG image for files
  llms.txt/
    route.ts                            # GET /llms.txt
```

### Cache Headers (raw file responses)
- Never-expiring bucket: `public, max-age=31536000, immutable`
- Expiring bucket: `public, max-age={seconds_remaining}`
- Expired resource: `no-store`, 410 Gone

### Expiry Cleanup
Primary: `instrumentation.ts` (`register()` export) starts a `setInterval` every 15 minutes on server startup. Deletes expired buckets from DB (cascade deletes `files` rows) and removes their directories from `$DATA_DIR/files/`.

Fallback (if instrumentation proves unreliable): Shell loop in Docker entrypoint (`while true; do node cleanup.js; sleep 900; done`) running alongside the Next.js process.

## API Conventions

### Response Shape
All responses include `url`, `api_url` (and `raw_url` for files):
```json
{
  "id": "aBcD1234eF",
  "name": "q1-reports",
  "owner": "SQL Buddy",
  "for": "Carbon",
  "created_at": 1708700000,
  "expires_at": 1709304800,
  "url": "https://clawd.example.com/aBcD1234eF",
  "api_url": "https://clawd.example.com/api/buckets/aBcD1234eF"
}
```

### Error Shape
```json
{
  "error": "Bucket not found",
  "hint": "Check the bucket ID in the URL. Buckets expire â€” it may have been deleted."
}
```

### API Endpoints

**Keys (admin only):**
- `POST /api/keys` â€” body: `{ name }` â†’ returns raw key (once), prefix, name
- `GET /api/keys` â€” lists all keys (prefix, name, created_at, last_used_at, bucket_count)
- `DELETE /api/keys/:key_prefix` â€” revokes by prefix. Buckets become read-only orphans.

**Buckets:**
- `POST /api/buckets` â€” body: `{ name, description?, for?, expires_in?, visibility? }` â†’ creates bucket
- `GET /api/buckets` â€” lists caller's non-expired buckets (filtered by key_hash)
- `GET /api/buckets/:id` â€” bucket info + flat file listing array
- `PATCH /api/buckets/:id` â€” update name, description, expires_at
- `DELETE /api/buckets/:id` â€” deletes bucket + all files from disk

**Files:**
- `POST /api/buckets/:id/upload` â€” multipart form data. Field names are file paths (e.g. `reports/q1.csv`). Supports uploading entire directory trees in one request. Re-uploading a path replaces the file.
- `DELETE /api/buckets/:id/files` â€” body: `{ path }` â†’ deletes single file

**ZIP:**
- `GET /api/buckets/:id/zip` â€” streams ZIP of entire bucket

**LLM-friendly:**
- `GET /api/buckets/:id/summary` â€” plain-text bucket summary (name, owner, file listing, README)
- `GET /llms.txt` â€” machine-readable platform overview (llms.txt convention)
- `GET /docs/api.md` â€” full API reference as raw markdown

## Pages

| Page | Route | Notes |
|---|---|---|
| Landing | `/` | Branding, feature overview, link to docs. No bucket listing. |
| Bucket view | `/:bucket_id` | File tree, README, metadata bar, ZIP download button |
| File preview | `/:bucket_id/*path` | Content-negotiated. Type-specific preview. |

### Page Details

**Landing (`/`):** Branding hero, feature cards, link to `/docs`. Instrument Serif headings, Geist Mono body. No bucket listing.

**Bucket view (`/[bucket]`):**
- Metadata bar: owner, "for" field, expiry countdown, file count, visibility badge
- GitHub-style file tree table (icon, name, size, modified date). Folders clickable to navigate.
- Breadcrumb navigation
- README.md rendered below file listing (if present)
- "Download ZIP" button

**File preview (`/[bucket]/[...path]`):**
- Top bar: filename, size, MIME type, expiry, breadcrumb to bucket, download button, raw link, curl command
- Body: type-specific preview
- Markdown files: client component toggle between rendered and raw source

**Admin dashboard (`/admin?token=...`):**
- Server component verifies HMAC token on each request
- Stats overview: total buckets, files, storage used
- API keys table with usage stats + revoke buttons (client component islands)
- Buckets table with expiry status + delete buttons
- Revoke/delete call API endpoints + `router.refresh()`

**API docs (`/docs`):**
- Static server-rendered page, Stripe-style section layout
- Each endpoint: method badge, path, description, request/response examples, copyable curl
- Raw markdown available at `/docs/api.md`

## Docker

Single container. Multi-stage build:
1. `deps` stage: `npm ci`
2. `build` stage: `next build` (standalone output)
3. `runner` stage: Node.js alpine, copy standalone output + `public` + `.next/static`

`DATA_DIR` mounted as a volume. `ADMIN_API_KEY`, `BASE_URL`, `PORT` set via environment. Instrumentation hook handles cleanup; shell loop fallback available in entrypoint if needed.
